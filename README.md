# Project: RHCSA_10-Task_Server_Configuration

## Project Context:

This project was built as a study plan moving forward to take the RHCSA EX200 Certification Exam. The main purpose of this project is mock the exam through a list of 10 different tasks to configure a server; going from user and group creation to resolve a SELinux incident. 

This simulation is designed to:

- Fell like the real exam.
- Force context switching
- Combine objectives.
- Test persistence and trouble shooting.

## Naming Conventions:

### Servers:

- servera.lab.local: Main server where most part of the configurations will be executed.
- serverb.lab.local: Secondary service used to test or for extra configurations.

## Cover Tasks:

### Servera.lab.local:

- User Administration.
- File Permissions.
- Archives & compression
- Software Management.
- Storage Management.
- Extend Storage.
- Firewall Configuration.
- SELinux Troubleshooting.
- Service Management.

### serverb.lab.local:

- Network Configuration.

---

## Repository configuration.

Following a Exam Scenario, for this project the internet access is not allowed, therefor a ISO disk has been attach to the VM in order to provide a local isolated repository for any installation required and its configuration is part of a tasks during the exam. 

---

## Project Rules:

Mocking a real world Exam Scenario this project is building keeping this rules in my:

- Perform all tasks as instructed
- Reboots are allowed and may be required
- Changes must persist after reboot
- Do not install additional software unless instructed
- Verify your own work
- No external internet access
- Use only local documentation and man pages

---

## Summary of Steps

This list of task are thinking to provide a practice way to be prepared for the actual exam. Targeting daily-base configurations and troubleshooting in any Linux server. 

Screenshots will be added to provide a visual verification of the configuration.
